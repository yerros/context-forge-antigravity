# Progress Tracker

Update this file after every meaningful implementation
change.

> **Keep this file lean — it is read on every resume/build.**
> Hold an *active window* only: current phase/goal, In
> Progress, Next Up, Open Questions, the ~10 most recent
> Completed units, and the ~8 most recent Session Notes.
> When it grows past that (or ~6 KB / ~1,500 tokens), move
> the oldest Completed entries and Session Notes into
> `context/progress-archive.md` (append newest-first). That
> archive is history and is **not** auto-read, so rotating
> it out saves tokens with no loss of active context.
